# Glossary

Key terms used throughout the Frankmax CMS documentation.

## A

**Audit Log** — An immutable record of all status changes, publishes, and ratifications. Entries cannot be updated or deleted once created.

**allBlocks** — The exported array from `src/blocks/index.ts` containing all 20 block definitions, used by the Pages collection `layout` field.

## B

**Block** — A Payload CMS Block type representing a reusable content component. The CMS includes 20 blocks organized into layout, content, media, and feed categories.

**BentoGrid** — A block type that renders a featured card spanning 2 columns alongside 2-6 smaller cards.

## C

**Campaign** — A time-bound content initiative with start/end dates, budget, KPI targets, and linked content items. Statuses: draft, scheduled, active, paused, completed.

**Content Filter** — A dynamically built Payload `Where` clause that applies jurisdiction targeting, language targeting, and scheduling date windows. Generated by `buildContentFilter()`.

**Content Hash** — A SHA-256 hash of a document's content fields (excluding metadata like status, timestamps, id). Used for integrity verification.

**Content Orchestrator** — A custom admin view for bulk operations across collections, including assigning jurisdictions and changing statuses.

## D

**Doctrine** — Governance documents organized into layers: Immutable Law, Constitutional Law, PEACE Protocol, and Operational Guideline. Can be made immutable after ratification.

**Draft** — The initial status of content before it enters the review workflow.

## G

**Governance Hook** — A `beforeChange` hook created by `createGovernanceHook()` that enforces status transitions, generates content hashes, creates audit log entries, and manages publish tokens.

## J

**Jurisdiction** — A country-specific configuration containing a country code, locale/region, language, theme colors, and status. Frankmax operates in 250+ jurisdictions.

## L

**Live Preview** — A Payload CMS feature that shows real-time content changes in an iframe with breakpoints for mobile (375px), tablet (768px), and desktop (1440px).

**Locale** — A URL path segment like `en-gb` or `fr-ca` combining a language code and country code. Used for routing and content targeting.

## M

**MeiliSearch** — The search engine used for full-text search across services, software, and news. Configured in `src/lib/search.ts`.

## P

**Payload CMS** — The headless CMS framework (v3) that powers the Frankmax admin panel, collections, and API.

**Publish Token** — An immutable record created when content is published, containing the content hash, publisher, and document reference. Used for integrity verification.

**PEACE Protocol** — One of the four doctrine governance layers.

## R

**Ratified** — A content status indicating the content has been formally approved. For doctrine, ratification records who ratified it and when.

**RenderBlocks** — The React component (`src/components/blocks/RenderBlocks.tsx`) that maps block types to their React component implementations.

## S

**Sovereign Theme** — A country-specific color palette derived from flag colors, applied via CSS custom properties. Defined by the `SovereignPalette` interface with 8 color tokens.

**SovereignPalette** — TypeScript interface defining the 8 theme colors: primary, secondary, accent, background, foreground, sidebarBg, sidebarFg, surface.

**Sub-brand** — One of 5 Frankmax Digital entities: FDS001 (Digital Marketing Maven), FDS002 (Design Studio), FDS003 (IT Synergy), FDS004 (DevOps Excellence), FDS005 (AI Labs).

## T

**Target Jurisdictions** — A reusable relationship field allowing content to target specific jurisdictions. When empty, content is treated as global.

**Target Languages** — A reusable select field with 10 language options (en, fr, es, de, pt, ar, zh, ja, hi, ru). When empty, content is shown for all languages.
