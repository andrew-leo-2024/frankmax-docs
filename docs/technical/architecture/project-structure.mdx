# Project Structure

The Frankmax CMS follows the standard Payload CMS v3 + Next.js App Router conventions with additional directories for blocks, custom admin views, and shared libraries.

## Directory Layout

## Key Files

### `payload.config.ts`

The root Payload configuration file. Defines all collections, globals, block registrations, admin panel customizations, database adapter, editor configuration, and plugin registrations.

### `src/blocks/index.ts`

Exports the `allBlocks` array containing all 20 block type definitions. This array is referenced by the Pages collection's `layout` field.

### `src/payload-types.ts`

Auto-generated TypeScript type definitions for all collections, globals, and blocks. Regenerated via `payload generate:types`.

### `src/lib/payload.ts`

Singleton factory for the Payload client instance. Used by server components and API routes to interact with the CMS.

## Conventions

| Convention | Details |
|-----------|---------|
| Collection files | PascalCase filenames (e.g., `NewsArticles.ts`) |
| Block files | PascalCase filenames matching the block slug |
| Component files | PascalCase filenames with `.tsx` extension |
| Library files | kebab-case filenames with `.ts` extension |
| Route groups | `(frontend)` for public routes, `(payload)` for admin/API |
| Locale format | `en-{country-code}` (e.g., `en-gb`, `en-us`) |
