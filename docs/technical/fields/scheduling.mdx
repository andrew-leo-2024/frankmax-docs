# Scheduling Fields

Two reusable date fields that provide time-based content visibility control: scheduled publication and automatic expiration.

**Source:** `src/collections/fields/scheduling.ts`

## Configuration

The scheduling module exports an array of two field definitions:

```typescript
// src/collections/fields/scheduling.ts
export const schedulingFields = [
  {
    name: 'scheduledPublishAt',
    type: 'date',
    admin: {
      date: {
        pickerAppearance: 'dayAndTime',
      },
      position: 'sidebar',
    },
    index: true,
  },
  {
    name: 'expiresAt',
    type: 'date',
    admin: {
      date: {
        pickerAppearance: 'dayAndTime',
      },
      position: 'sidebar',
    },
    index: true,
  },
];
```

## Fields

### scheduledPublishAt

| Property | Value |
|----------|-------|
| **Name** | `scheduledPublishAt` |
| **Type** | date |
| **Picker** | `dayAndTime` |
| **Indexed** | `true` |
| **Admin Position** | sidebar |

Controls when content becomes visible on the frontend. Content with a `scheduledPublishAt` date in the future will not appear in query results.

### expiresAt

| Property | Value |
|----------|-------|
| **Name** | `expiresAt` |
| **Type** | date |
| **Picker** | `dayAndTime` |
| **Indexed** | `true` |
| **Admin Position** | sidebar |

Controls when content automatically stops appearing on the frontend. Content with an `expiresAt` date in the past will not appear in query results.

## Content Filter Integration

The `buildContentFilter()` function generates scheduling conditions by default (controlled by the `scheduling` option, which defaults to `true`):

```typescript
// Scheduling conditions generated by buildContentFilter()
{
  and: [
    {
      or: [
        { scheduledPublishAt: { less_than_equal: now } },
        { scheduledPublishAt: { exists: false } },
      ],
    },
    {
      or: [
        { expiresAt: { greater_than: now } },
        { expiresAt: { exists: false } },
      ],
    },
  ],
}
```

This ensures:

- Content without `scheduledPublishAt` is immediately visible
- Content with `scheduledPublishAt` in the past or present is visible
- Content without `expiresAt` never expires
- Content with `expiresAt` in the future is still visible
- Content with `expiresAt` in the past is hidden

See [Content Filters](../libraries/content-filters) for details.

## Indexing

Both fields are indexed in the database for efficient time-based queries, since scheduling checks run on every frontend page load.

## Admin UI

Both fields appear in the sidebar with a date-and-time picker interface, allowing editors to set precise publication and expiration timestamps.
