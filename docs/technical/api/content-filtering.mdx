# Content Filtering

The Payload REST API supports powerful filtering through `where` query parameters. The Frankmax CMS uses these extensively for jurisdiction-based content delivery and scheduling.

## Where Clause Syntax

### Basic Equality

```
GET /api/pages?where[status][equals]=published
```

### Comparison Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `equals` | Exact match | `where[status][equals]=published` |
| `not_equals` | Not equal | `where[status][not_equals]=draft` |
| `greater_than` | Greater than | `where[price][greater_than]=100` |
| `greater_than_equal` | Greater than or equal | `where[priority][greater_than_equal]=1` |
| `less_than` | Less than | `where[price][less_than]=5000` |
| `less_than_equal` | Less than or equal | `where[scheduledPublishAt][less_than_equal]=2026-03-01` |
| `like` | Pattern match (SQL LIKE) | `where[name][like]=cloud` |
| `contains` | Contains value | `where[targetLanguages][contains]=en` |
| `in` | In array of values | `where[status][in]=draft,review` |
| `not_in` | Not in array | `where[status][not_in]=draft` |
| `exists` | Field exists/not null | `where[targetJurisdictions][exists]=true` |

### Relationship Filtering

Filter by relationship field values:

```
GET /api/pages?where[targetJurisdictions][equals]=jurisdiction-id-123
```

### Combining with AND

Use array syntax for AND conditions:

```
GET /api/pages?where[and][0][status][equals]=published&where[and][1][targetJurisdictions][equals]=123
```

### Combining with OR

```
GET /api/pages?where[or][0][status][equals]=draft&where[or][1][status][equals]=review
```

### Nested AND/OR

```
GET /api/pages?where[and][0][status][equals]=published&where[and][1][or][0][targetJurisdictions][exists]=false&where[and][1][or][1][targetJurisdictions][equals]=123
```

## Common Filtering Patterns

### Published Content for a Jurisdiction

```
GET /api/pages?where[and][0][status][equals]=published&where[and][1][or][0][targetJurisdictions][exists]=false&where[and][1][or][1][targetJurisdictions][equals]={jurisdictionId}
```

### Content by Language

```
GET /api/services?where[or][0][targetLanguages][exists]=false&where[or][1][targetLanguages][contains]=en
```

### Active Campaigns

```
GET /api/campaigns?where[status][equals]=active
```

### News by Jurisdiction

```
GET /api/news-articles?where[jurisdiction][equals]={jurisdictionId}
```

### Scheduled Content (Not Yet Published)

```
GET /api/pages?where[scheduledPublishAt][greater_than]={now}
```

### Non-Expired Content

```
GET /api/pages?where[or][0][expiresAt][exists]=false&where[or][1][expiresAt][greater_than]={now}
```

## buildContentFilter() Equivalent

The `buildContentFilter()` function generates Where clauses programmatically that are equivalent to the REST API syntax above. Here is the mapping:

### Programmatic Usage

```typescript
import { buildContentFilter } from '@/lib/content-filters';

const filter = buildContentFilter({
  jurisdictionId: '123',
  lang: 'en',
  scheduling: true,
});

// Use with Payload client
const pages = await payload.find({
  collection: 'pages',
  where: {
    ...filter,
    status: { equals: 'published' },
  },
});
```

### REST API Equivalent

The filter generated by `buildContentFilter({ jurisdictionId: '123', lang: 'en', scheduling: true })` is equivalent to this REST query:

```
GET /api/pages?where[and][0][or][0][targetJurisdictions][exists]=false
              &where[and][0][or][1][targetJurisdictions][equals]=123
              &where[and][1][or][0][targetLanguages][exists]=false
              &where[and][1][or][1][targetLanguages][contains]=en
              &where[and][2][or][0][scheduledPublishAt][less_than_equal]=2026-03-01T00:00:00Z
              &where[and][2][or][1][scheduledPublishAt][exists]=false
              &where[and][3][or][0][expiresAt][greater_than]=2026-03-01T00:00:00Z
              &where[and][3][or][1][expiresAt][exists]=false
              &where[status][equals]=published
```

See [Content Filters Library](../libraries/content-filters) for the full function documentation.

## Sorting

Sort results using the `sort` parameter:

```
GET /api/services?sort=name           # Ascending by name
GET /api/services?sort=-createdAt     # Descending by creation date
GET /api/campaigns?sort=startDate     # Ascending by start date
```

## Pagination

Control pagination with `limit` and `page`:

```
GET /api/news-articles?limit=20&page=2
```

Use `limit=0` to get only the `totalDocs` count without returning any documents (useful for analytics).
